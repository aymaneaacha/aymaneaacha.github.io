Medical Dataset EDA using KNIME

Overview

This project involved using the KNIME platform to conduct exploratory data analysis (EDA) and data preprocessing on a medical dataset containing patient records related to heart disease. The focus was on ethical data cleaning, ensuring statistical integrity, and preparing the dataset for future predictive analysis.

Key Actions

  • Imported and renamed columns using KNIME nodes (CSV Reader, Column Renamer).
  • Identified attributes with significant missing data: slope (33.6%), ca (66.4%), and thal (52.8%).
  • Removed columns and rows with excessive missing values due to ethical concerns around imputation in medical data.
  • Eliminated impossible values such as cholesterol and blood pressure readings of 0.
  • Retained numerical outliers to avoid potential statistical bias (e.g., type I errors).
  • Confirmed all remaining records were unique using Duplicate Row Filter node.

Conclusions

After careful evaluation and cleaning using KNIME workflows, the dataset was reduced from 920 to 661 records, but the data integrity was preserved. This thorough preprocessing ensured the dataset is well-suited for further analysis or model development, especially important in sensitive domains like healthcare.
